Performs the product of a Toeplitz matrix by a general matrix using the sliding window algorithm.

This is a simplifed call of the sequential product including initialization and cleaning. It uses the flag parameters to select a computation mode.

Parameters:

na number of rows of V;
ma number of columns of V;

Ta Toeplitz matrix stored as a vector of a length lambda and composed of only non-zero entries of its first row
lambdanumber of non-zero in the first row of the Toeplitz and size of T

Performs the multiplication of a symmetric, Toeplitz block-diagonal matrix, T, by an arbitrary data matrix, V.

The product is computed block-by-block each time applying the sliding window algorithm. The diagonal blocks are assumed to be symmetric, band-diagonal Toeplitz matrices but which can differ from a block to a block. The parameters are :

Parameters:
V[input] arbitrary data matrix stored in a column-wise order, i.e., V(i,j) = V[ i+j*n];
V[out] result of the product T x V;
nrowa number of rows of the data matrix V;
ma number of columns of V;
tpltzblocksa list of the toeplitz blocks constituting the matrix T. Each block is characterized byits own set of parameters (idv, n, T_block, lambda) :
  • idv a global row index defining for each Toeplitz block a number of the first sample of the time interval to which this Toeplitz matrix applies;
  • n a dimension of each Toeplitz block;
  • T_block a vector storing all non-zero (= lambda) entries of the first row of the block;
  • lambda defines the size of the Toeplitz block diagonal band, with the full bandwith equal to lambda*2-1;
nb_blocks_alla number of all diagonal Toeplitz block;
idpan index of the first element of V which is to be taken into account;
flag_stgyflag strategy for the product computation

Performs the multiplication of a symmetric, Toeplitz block-diagonal matrix with gaps, T, by an arbitrary matrix, V, distributed over processes.

We first rebuild the Toeplitz block matrix structure to reduce the computation cost and skip the computations of the values on the defined gaps.

The parameters are :

Parameters:
V[input] distributed data matrix (with the convention V(i,j)=V[i+j*n]) ; [out] result of the product TV
nrownumber of rows of the global data matrix V
mnumber of columns for the data matrix V in the global rowwise order
m_rowwisenumber of columns for the data matrix V in the rowwise order per processor
tpltzblockslist of the toeplitz blocks struture with its own parameters (idv, n, T_block, lambda) :
  • idv is the global row index defining for each Toeplitz block as stored in the vector T ;
  • n size of each Toeplitz block
  • T_block location of each Toeplitz matrix data composed of the non-zero entries of the first row ;
  • lambda size of each Toeplitz block data T_block. The bandwith size is then equal to lambda*2-1
nb_blocks_allnumber of all Toeplitz block on the diagonal of the full Toeplitz matrix
nb_blocks_localnumber of Toeplitz blocks as stored in T
idpglobal index of the first element of the local part of V
local_V_sizea number of all elements in local V
id0gapindex of the first element of each defined gap
lgaplength of each defined gaps
ngapnumber of defined gaps
flag_stgyflag strategy for the product computation
